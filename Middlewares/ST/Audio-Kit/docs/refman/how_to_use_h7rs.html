How to Use STM32H7S78-DK

The STM32H7S78-DK board embeds a STM32H7S7L8 MCU. Being a flash-less device, the STM32H7S78-DK requires to flash a boot loader before flashing the application. Please refer to the reference manual section 2.6 Boot configuration. In this project, we use the BOOT0 set to 0 meaning that the bootloader hex file delivered in this package will boot from the user flash memory at 0x0800000.

STM32H7S78-DK Specificities and constraints

The memory available is :

  • internal FLASH : 64 Kbytes
  • internal SRAM : 620 Kbytes
  • external FLASH : 128 MBytes of Octo xSPI NOR flash memory
  • external RAM : 32 MBytes of Hexadeca xSPI PSRAM

LiveTune Memory Map

0x10 0xFFFF 64 KB ITCM application (data) from 0x10 to avoid coherent NULL pointers (tricky to debug)
0x08000000 0x0800FFFF 64 KB FLASH Bootloader
0x20000000 0x2000FFFF 64 KB DTCM application (data)
0x24000000 0x2401FFFF 128 KB AXI-SRAM1 application (data)
0x24020000 0x2403FFFF 128 KB AXI-SRAM2 application (data)
0x24040000 0x2405FFFF 128 KB AXI-SRAM3 application (data)
0x24060000 0x24069FFF 40 KB AXI-SRAM4 application (data). This is AXI-SRAM shared with ECC.
0x2406A000 0x24071FFF 32 KB AXI-SRAM4 Non cached zone for HW (DMA buffers). This is AXI-SRAM shared with ECC.
0x30000000 0x30007FFF 32 KB AHB SRAM Not used
0x38800000 0x38800FFF 4 KB BCK SRAM Not used
0x70000000 0x77FFFFFF 128 MB NOR FLASH Application binary + registry
0x90000000 0x91FFFFFF 32 MB PSRAM Application (ro) + memory pool

Implementation

  • The internal FLASH hosts the bootloader code.

  • The internal RAM is used by the application (rw data + heap)

  • The external RAM is split into two parts :

    • one part hosts the application code (ro data + code executed by application).
    • a second part is used as memory pool by the application (extends the internal memory)

  • The external FLASH is split into two parts:

    • The first part contains the application binary (copied by the bootloader to external RAM) before execution.
    • The second part contains the application registry (accessed by the application).

Flashing Binaries

Please locate the Bin folder in your X-CUBE-AUDIO Kit package.

In this folder, please locate the Fast & Secure Boot Loader called X-CUBE-AUDIO-KIT-STM32H7S78-DK-BL.bin. This boot loader will copy whatever firmware is loaded into RAM. Therefore, in order to flash your application, follow these steps:

  1. Make sure the BOOT0 is in the right position as explained above.
  2. Flash X-CUBE-AUDIO-KIT-STM32H7S78-DK-BL.bin in order to boot from internal flash (bootloader)
    • bootloader configures external FLASH and RAM
    • bootloder copies Application from external FLASH to external PSRAM
    • bootloder deinit external FLASH
    • bootloader jumps to application in external PSRAM
  3. Flash the application firmware.
    • application re-initializes external FLASH in order to be used by application (registry)
  4. Reset the board if needed.

In Bin, please locate the ac_flash_firmware.sh flasher. This script is given as an example to help the flashing process:

  • It locates all the binaries in a given folder.
  • Asks the user which one should be flashed.
  • Flashes the requested binary (boot loader or application).

A similar script is also provided inside the application folder to do the same using a freshly compiled binary; the flasher.sh file.

STM32CubeIDE Tips

After the board is flashed, click on the Debug button. Please use the debug configuration provided in the firmware.launch file. It attaches to the STM32 that is running. Setting breakpoint that are called by the running application is possible. However, breaking in the init phase of the application isnt. This limitation is under investigation.
